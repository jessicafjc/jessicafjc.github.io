环境配置

  • 查看本机Python版本python -V，我的默认版本是2.7，去Python官网下载一个3.7的版本进行安装。

  • 安装完成后，修改默认启动版本为3.7版本，打开open ~/.bash_profile，添加alias python="/usr/local/bin/python3.7"，不要忘记source ~/.bash_profile重新加载一下。再次python -V发现Python成功变成3.7版本。

  • 前往Anaconda官网下载对应的安装包，进行安装。

  • 安装Tensorflow，先查看是否有conda --version，有可能报错conda command not found

    • 先检查anaconda中的bin文件夹内是否有（有）

    • 在vi ~/.zshrc中添加`export PATH=/Users/jiachenfu/anaconda3/bin:$PATH

    • 然后source ~/.zshrc，检查一下conda --version成功

    • 更改下载源为清华镜像conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/，然后conda config --set show_channel_urls yes

    • 最后安装Tensorflowconda install tensorflow

    • 打开Anaconda中的jupyter,新建一个Python3文件,运行

      import tensorflow as tf 
tf.__version__

      成功输出1.1.0

Vue初探(一)安装及项目创建

Posted on 2019-11-28 | In Web

Vue了解

渐进式JavaScript 框架

Vue开发环境配置

  1. Vue依托nodejs，vue只是node万千包中的一个，所以首先要安装nodejs

  2. 查看是否安装node，可以使用node -v版本号查看，若未安装下载地址

  3. 查看是否安装vue vue -V注意大写

  4. 安装Vue npm install -g @vue/cli

    • 前面加sudo 是获取系统的权限，之后需要输入密码，不加sudo可能因为权限的问题安装失败
    • 如果安装vue出现一些警告和错误，清除一下缓存就行 npm cache clean --force

  5. 3.0以上版本集成了图形界面，所以卸载2.9.6等旧版本的命令：sudo npm uninstall vue-cli -g

    创建并配置Vue项目

  6. 创建或前往本地文件夹，这里以新创建文件夹 VueProject为例mkdir VueProject

  7. 创建项目vue create 项目名，项目名必须小写

  8. 上下键选择，enter键确认，我这里选择Manually

  9. 空格键选中，a键全选，enter键提交，以下为简单描述

  10. 上一步我尝试全部选择，按键a，然后enter键入，进一步配置

  • vue3提供了一个图形化创建项目的方法vue ui

启动项目

  1. 前往项目文件夹cd 项目文件夹
  2. 启动项目npm run serve

vue的命令

  • init 从模板生成新项目
  • list 列出可用的官方模板
  • build 创建新项目的原型
  • create 创建（警告：仅用于v3以上版本）
  • vue ui 启动脚手架图形界面（3.0以上版本）

常用命令

  • npm install 安装依赖
    • npm install mysql //连接mysql数据库的依赖
  • npm run dev 或 npm run serve 启动项目

ssh远程连接–>sftp

  • 配置实例安全组，开放22端口
    ssh 如果连接不成功，提示ssh_exchange_identification: read: Connection reset by peer

解决办法

  1. 前往本地文件夹/Users/jiachenfu/.ssh/删除known_hosts或改为known_hosts.bak，尝试ssh root@ip是否解决
  2. 切换一个网络吧，关闭代理，有的校园网络很坑会连不上
  3. 尝试ping ip，如果成功，前往阿里云网页进行远程登录，修改vi /etc/hosts.allow，在末尾添加sshd: ALL ##允许所有ip主机均能连接本机，然后重启systemctl restart sshd.service，尝试ssh root@ip是否解决
  4. 查看阿里云实例密钥对是否绑定

Linux/Mac OS常用指令

Posted on 2019-11-25 | In Liunx

常用指令

  • 查看当前地址pwd
  • 查看当前位置内所有文件ls
  • 查看当前位置内所有文件及详细信息ls -s
  • 前往某地址cd 前往的地址目录
  • 回到上一级目录 cd ..回到上上级cd ../..
  • 新建文件夹 mkdir 文件夹名
  • 断开连接ctl + c

vim编辑器

vim几种模式

  • 正常模式：可以使用快捷键命令，或按:输入命令行。
  • 插入模式：可以输入文本，在正常模式下，按i、a、o等都可以进入插入模式。
  • 可视模式：正常模式下按v可以进入可视模式， 在可视模式下，移动光标可以选择文本。按V进入可视行模式， 总是整行整行的选中。ctrl+v进入可视块模式。
  • 替换模式：正常模式下，按R进入。

vim常用指令

  • 行号显示
  • 关键词查找
  • 修改i
  • 保存退出:wq
  • 在后面的文本中查找/
  • ?something: 在前面的文本中查找something。
  • /pattern/+number: 将光标停在包含pattern的行后面第number行上。
  • /pattern/-number: 将光标停在包含pattern的行前面第number行上。
  • n: 向后查找下一个。
  • N: 向前查找下一个。

搭建hexo个人博客

Posted on 2019-11-25 | In Web

环境搭建

  1. node.js安装官网
  2. npm或cnpmnpm install -g cnpm --registry-https://registry.npm.taobao.org
  3. 验证版本node -v``npm -v或cnpm -v
  4. 安装hexocnpm install -g hexo-cli

创建本地Blog

  1. 建立空文件夹作为博客地址，mkdir Blog
  2. 前往该文件夹 cd Blog/
  3. 初始化hexo hexo init 这里使用命令创建文件夹是因为我通过cmd + N方式创建文件夹会提示为非空导致无法初始化hexo
  4. 启动hexo 服务 hexo s 成功后可访问http://localhost:4000/，使用ctl+C结束服务
  5. 创建一篇Blog hexo n "文件名"，生成的Blog是markdown格式，会在source/_posts/这个文件夹下
  6. 清理缓存hexo clean，生成博客 hexo g
  7. 本地启动hexo测试效果 hexo s

部署至Github个人主页

  1. 设置Github的个人主页

  2. 在Blog目录下安装git部署插件 cnpm install --save hexo-deployer-git

  3. 在博客配置文件中修改信息 vim _config.yml 在最底部添加：

  4. 部署到GitHub hexo d，访问https://GitHub用户名.github.io/查看效果

更改主题

  1. 在网上挑选好看的hexo主题
  2. 通过git clone克隆到本地Blog中的theme文件夹内 git clone https://github.com/iissnan/hexo-theme-next themes/next
  3. 修改博客配置文件 vim _config.yml
    • 更改主题 theme: 主题文件夹名称
    • 更改头像也在这里，找这个：avatar:
  4. 清理hexo clean然后hexo d -g，或者本地查看效果hexo s

自定义主题配置

  1. 前往博客地址 cd Blog/
  2. 修改主题配置文件 vim themes/next/_config.yml
    • 修改样式查找关键词scheme:
    • 修改侧边栏查找关键词menu:
      • 需要添加单独页面hexo new page "tags"
  3. 保存后清理hexo clean，本地运行debug模式测试hexo s --debug

遇到的坑

  • 使用sudo su即系统角色进行操作后无法再使用管理者或普通用户修改文件
    • 严格的权限管理是Liunx系统的优点，所以在进行hexo配置时需要保证权限统一，若平时使用普通用户或管理者权限进行blog创作，请在hexo init及后续操作时不要使用sudo su
  • 修改menu后找不到对应页面，提示Cannot GET XXX/%20/
    • 在主题配置文件中找到menu:对应内容，将所需功能的/ ||之间的空格删除
  • 修改主题后部署到GitHub上主题不改变
  • 先清理hexo clean，后部署生成hexo d -g，主题改变可能需要等一点时间，不会立刻修改完毕
