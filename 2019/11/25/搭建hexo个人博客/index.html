搭建hexo个人博客

Posted on 2019-11-25 Edited on 2020-05-21 In 散装知识

环境搭建

  1. node.js安装官网
  2. npm或cnpmnpm install -g cnpm --registry-https://registry.npm.taobao.org
  3. 验证版本node -v``npm -v或cnpm -v
  4. 安装hexocnpm install -g hexo-cli

创建本地Blog

  1. 建立空文件夹作为博客地址，mkdir Blog
  2. 前往该文件夹 cd Blog/
  3. 初始化hexo hexo init 这里使用命令创建文件夹是因为我通过cmd + N方式创建文件夹会提示为非空导致无法初始化hexo
  4. 启动hexo 服务 hexo s 成功后可访问http://localhost:4000/，使用ctl+C结束服务
  5. 创建一篇Blog hexo n "文件名"，生成的Blog是markdown格式，会在source/_posts/这个文件夹下
  6. 清理缓存hexo clean，生成博客 hexo g
  7. 本地启动hexo测试效果 hexo s

部署至Github个人主页

  1. 设置Github的个人主页

  2. 在Blog目录下安装git部署插件 cnpm install --save hexo-deployer-git

  3. 在博客配置文件中修改信息 vim _config.yml 在最底部添加：

  4. 部署到GitHub hexo d，访问https://GitHub用户名.github.io/查看效果

更改主题

  1. 在网上挑选好看的hexo主题
  2. 通过git clone克隆到本地Blog中的theme文件夹内 git clone https://github.com/theme-next/hexo-theme-next themes/next已更新V7版本，V5.1.4及以前版本地址为 git clone git clone https://github.com/theme-next/hexo-theme-next themes/next
  3. 修改博客配置文件 vim _config.yml
    • 更改主题 theme: 主题文件夹名称
    • 更改头像也在这里，找这个：avatar:
  4. 清理hexo clean然后hexo d -g，或者本地查看效果hexo s

自定义主题配置

  1. 前往博客地址 cd Blog/
  2. 修改主题配置文件 vim themes/next/_config.yml
    • 修改样式查找关键词scheme:
    • 修改侧边栏查找关键词menu:
      • 需要添加单独页面hexo new page "tags"
  3. 保存后清理hexo clean，本地运行debug模式测试hexo s --debug

遇到的坑

  • 使用sudo su即系统角色进行操作后无法再使用管理者或普通用户修改文件
    • 严格的权限管理是Liunx系统的优点，所以在进行hexo配置时需要保证权限统一，若平时使用普通用户或管理者权限进行blog创作，请在hexo init及后续操作时不要使用sudo su
  • 修改menu后找不到对应页面，提示Cannot GET XXX/%20/
    • 在主题配置文件中找到menu:对应内容，将所需功能的/ ||之间的空格删除
  • 修改主题后部署到GitHub上主题不改变
  • 先清理hexo clean，后部署生成hexo d -g，主题改变可能需要等一点时间，不会立刻修改完毕
  • 侧栏中的posts打开的链接错误，404页面，访问链接显示/archives/%7C%7C%20archive/
    • 这是5.1.4版本bug，更新的7.0版本就好了；
